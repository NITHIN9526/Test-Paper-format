# 📄 QP Editor – Question Paper Format Builder

A professional, high-performance web application designed for educators to easily create, format, and generate academic question papers. This tool provides a seamless "What You See Is What You Get" (WYSIWYG) experience with a live A4 paper preview.

## ✨ Key Features

- **🚀 Live A4 Preview:** See exactly how your question paper will look on paper as you edit.
- **📐 Rich Content Support:**
  - **LaTeX Equations:** Render complex mathematical formulas using MathJax.
  - **Figures & Images:** Upload and align diagrams with custom captions and widths.
  - **Code Blocks:** Highlight pseudocode or programming snippets.
- **💾 Advanced Persistence:**
  - **Auto-Save:** Never lose work; changes are saved to LocalStorage automatically.
  - **Export/Import JSON:** Save your paper structure to a portable `.json` file and reload it anytime.
- **↗️ Professional PDF Export:** Generate print-ready A4 PDFs with one click.
- **↕️ Dynamic Reordering:** Easily move questions or entire parts up and down to restructure your exam.
- **🎨 Premium Dark UI:** A sleek, modern editor interface designed for focus and productivity.

## 🛠️ Technology Stack

- **Core:** HTML5, Vanilla JavaScript (ES6+), Vanilla CSS (Custom Design System)
- **Mathematical Rendering:** [MathJax v3](https://www.mathjax.org/)
- **PDF Generation:** [html2pdf.js](https://ekoopmans.github.io/html2pdf.js/)
- **Typography:** Google Fonts (Inter, Outfit, Times New Roman for Print)

## 🚀 Getting Started

1. **Clone the repository:**
   ```bash
   git clone https://github.com/NITHIN9526/Test-Paper-format.git
   ```
2. **Open the editor:**
   Simply open `index.html` in any modern web browser.
3. **Draft your paper:**
   Fill in the header details, add parts, and insert questions.
4. **Download:**
   Click **Download PDF** to get your print-ready document.

## 📂 Project Structure

- `index.html` - Application structure and libraries.
- `styles.css` - Premium dark UI and A4 print media queries.
- `script.js` - Core logic for state management, persistence, and rendering.
